### PPT生成工作流（修订版）

> **重要说明**：本需求文档已根据设计文档 `docs/superpowers/specs/2026-04-21-ppt-report-design.md` 进行修订。原方案中的服务器端文件系统操作已调整为纯浏览器端实现，工作流由 7 步精简为 3 步，并采用分阶段渐进交付策略。

开发一个PPT报告撰写的工作流，嵌入到现有的工作台项目中实现工作自动化：

#### 一、模板设置
在系统的左侧导航栏中增加一个模块，名为“撰写PPT”。

#### 二、工作流程（3步向导）

以下 3 个步骤在单页向导中串行完成，通过可视化进度条展示当前进度：

1. **填写报告信息**：用户填写报告名称、部门、日期和作者。日期默认当天，作者默认当前登录用户。填写完成后点击“下一步”。

2. **选择模板并配置内容**：
   - **选择PPT模板**：从系统预设的 3 套模板（品牌蓝、浅灰商务、深色高端）中选择一套。模板配置遵循 `/docs/ppt-design-system.md` 设计规范。
   - **选择数据来源**：从现有业务模块（销售收入、客户新增、宽带新增、智家产品、权益产品、家庭组网、周四复盘）中选择一个，系统自动调用对应数据生成器生成页面列表。
   - **编辑每页内容**：系统预填充每页的标题、结论和备注。用户可对标题（最多20字）、结论（最多150字）、备注（最多40字）进行手工修改，实时显示字数统计，超限时禁止进入下一步。
   - 提交后进入下一步。用户可点击进度条中已完成的步骤回退修改。

3. **生成预览视图并导出**：
   - 系统从上到下分页展示完整的 PPT 内容预览，每页遵循 `/docs/ppt-design-system.md` 的 16:9 标准尺寸和五区布局（标题区、结论区、内容区、备注区、页码区）。
   - 用户确认内容无误后，点击“导出PDF”，系统自动生成 PDF 文档并下载到本地。下载完成后报告自动存档到浏览器本地存储中。
   - 文件名格式：`{报告名称}-{部门}-{日期}-{作者}.pdf`

#### 三、PPT模板视图

遵循 `/docs/ppt-design-system.md` 设计规范：

1. **页面布局**：分标题区（上）、结论区（中上）、内容区（中下）、备注区（下）、页码区（右下角）
   - 标题区：位于PPT页面的上方，左对齐，最多不超过20字
   - 结论区：位于标题区的下方，左对齐，最多不超过150字，分2-3行显示
   - 内容区：位于结论区的下方，左对齐，根据画布的大小，生成上下、左右、上中下、左中右结构
   - 备注区：位于页面最下方，左对齐，用12pt字号备注，最多不超过40字，一行显示
   - 页码区：位于页面右下角，显示当页的页码

2. **内容组件**：提供标题文本框、内容文本框、各类 EChart 图表、表格组件

3. **PPT结构设计**：遵循 `/docs/ppt-design-system.md` 设计规范
   - PPT画布大小：默认为16:9的PPT标准尺寸

#### 四、功能参数

1. **工作流进度条**：进入工作流后，系统提供可视化步骤节点展示。3 个节点分别为“填写信息”、“配置内容”、“预览导出”。已完成的步骤显示绿色对勾，当前步骤蓝色高亮，未完成步骤灰色显示。用户可点击已完成的步骤回退修改。

2. **报告存档管理**：
   - 报告列表页展示所有历史报告，支持查看、重新编辑和删除。
   - 无报告时显示空状态并引导新建。

3. **模型配置（第三阶段启用）**：
   - 系统提供大模型接口参数配置入口，由用户自行输入 API Key（如 OpenAI）。
   - API Key 仅存储在用户浏览器本地（localStorage），不会上传至任何服务器。
   - 启用后，Excel 数据导入可调用大模型生成分析结论，或对用户输入的结论进行润色扩写。

4. **PPT模板设置（第四阶段启用）**：
   - 提供模板设计器，用户可上传自定义背景图、调整背景颜色、标题颜色等。
   - 自定义模板存储在浏览器本地，可在模板选择中使用。

#### 五、存储与命名规范

由于本项目为纯前端实现，所有数据存储在浏览器本地：

1. **报告数据存储**：使用浏览器 `localStorage`，键名为 `dashboard-ppt-reports`
2. **文件命名规范**：`{报告名称}-{部门}-{日期}-{作者}.pdf`
3. **模板存储**：第一阶段模板为内置 TypeScript 常量；第四阶段用户自定义模板存储在 `localStorage` 中

#### 六、分阶段开发路线图

| 阶段 | 周期 | 核心功能 | 说明 |
|------|------|---------|------|
| **第一阶段（MVP）** | 约 2 周 | 3 步向导 + 模板选择 + 数据生成 + PDF 导出 | 复用现有 `pdfExport.ts`、`SlideSection.tsx`、`slideSummary.ts` 基础设施 |
| **第二阶段（数据增强）** | 约 1-2 周 | Excel 导入 + sheet 到页面映射 + 表格数据填充 | 复用现有 `xlsx` 依赖解析 Excel |
| **第三阶段（AI 增强）** | 约 1-2 周 | 大模型配置 + 自动分析结论生成 + 单页润色 | 用户自行配置 API Key，可选启用 |
| **第四阶段（编辑器增强）** | 约 2 周 | 页面拖拽排序 + Markdown 富文本编辑器 + 自定义模板设计器 | 可选阶段，视前序阶段反馈决定 |

#### 七、与现有基础设施的复用

本模块最大化复用现有代码，避免重复建设：

- **PDF 导出**：复用 `src/utils/pdfExport.ts`（html2canvas → jsPDF，已处理 canvas 替换、暗黑模式、布局冻结）
- **PPT 页面渲染**：复用 `src/components/common/SlideSection.tsx`（标题区、结论区、内容区、页码区）
- **数据生成**：复用 `src/utils/dataGenerator.ts` 中各模块数据生成函数
- **结论生成**：复用 `src/utils/slideSummary.ts` 中各模块摘要函数
- **设计规范**：复用 `docs/ppt-design-system.md` 色彩、字体、字号、布局规范
- **UI 组件**：复用 `ModuleLayout.tsx`、`ExportButton.tsx`
